## xss

跨站脚本攻击

```
XSS 攻击分类：

1. 反射型 XSS：攻击者通过构造 URL 参数或表单提交数据来注入恶意脚本，受害者访问时即可被执行。

2. 存储型 XSS：攻击者将恶意脚本存储到服务器端，并等待受害者访问包含该脚本的页面时执行。

3. DOM 型 XSS：攻击者通过修改文档对象模型(DOM)来注入恶意脚本，受害者访问时即可被执行。

XSS 防护方法：

1. 输入过滤：对于所有用户输入的数据，均需要进行严格的输入验证和过滤，过滤掉其中的特殊字符、标签等，确保输入的内容是安全的。

2. 输出编码：对于输出到 HTML 页面的任何内容，包括从数据库中读取的内容，都需要进行 HTML 编码以防止被解析成恶意代码。

3. HttpOnly Cookie：设置 HttpOnly 属性禁止 JavaScript 访问 Cookie，这样攻击者就无法通过 XSS 攻击来窃取 Cookie 中的数据。

4. Content Security Policy(CSP)：使用 CSP，网站管理员可以指定浏览器只能从特定来源加载特定类型的资源，从而有效地减少 XSS 攻击的风险
```



## csrf



```
CSRF（Cross-Site Request Forgery）是另一种常见的 Web 安全漏洞，攻击者可以利用它来伪造用户的请求，以实施恶意操作。

CSRF 攻击原理：攻击者通过某些手段先让用户完成某些操作，如点击链接、访问网站等，然后在用户不知情的情况下，以用户身份向服务器发送请求，达到攻击的目的。

CSRF 防御方法：

1. 验证码：在关键操作前增加验证码，防止非正常用户恶意操作。

2. Token：为每个用户生成独特的 Token，且只有在用户进行操作时才会发送并验证，防止攻击者伪造请求。

3. 随机参数：在 URL 中添加随机参数，确保每个请求的唯一性，防止攻击者重放用户的请求。

4. Referer 检查：检查请求来源是否合法，可减少 CSRF 攻击的风险。但需注意，该方法容易被伪造，不可作为唯一的防御手段。

综上所述，XSS 和 CSRF 攻击是 Web 开发中常见的安全问题，需要开发人员采取多重防护手段来确保网站的安全性。
```
